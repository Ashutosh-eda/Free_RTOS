# RTOS Task Scheduling – Distilled Notes

## 1. Time Slicing
- CPU time divided into small units called **ticks**
- Hardware timer generates periodic interrupts

## 2. Scheduler Operation
At each tick:
1. Scheduler is triggered
2. Ready tasks are evaluated
3. Highest-priority task is selected

## 3. Priority-Based Scheduling
- Each task has a priority
- Higher priority tasks run first
- Lower priority tasks run only if higher ones are not ready

## 4. Task Execution Example
- Task A runs
- Calls `vTaskDelay()` → moves to Blocked state
- Resumes after delay expires

## 5. Idle Task
- Runs when no other task is ready

## 6. Round Robin Scheduling
- Same priority tasks share CPU time
- Executed alternately at each time slice

## 7. Task States
- **Running** → executing
- **Ready** → waiting for CPU
- **Blocked** → waiting for event/delay
- **Suspended** → paused manually

## 8. Blocking
vTaskDelay()
- Moves task: Running → Blocked
- Returns to Ready after delay

## 9. Suspend/Resume
vTaskSuspend()
vTaskResume()

- Used to control task execution manually

## 10. Context Switching
Scheduler saves/restores:
- Program Counter (PC)
- CPU registers
- Stack pointer

## 11. Preemptive Scheduling
- Higher-priority tasks interrupt lower-priority ones
- Task execution appears interleaved
